# Application Detail Ledger

Use this only when repeated job-application fields become part of the request.

Keep the first version as a local note/template, not a database or browser automation.

## Stash Shape

```text
Job:
Company:
Location:
Work type:
Pay / range:
Schedule:
Resume version:
Cover note needed:
Application URL:
Repeated fields:
Potential conflict:
Next action:
Logged:
```

## Typical Repeated Fields

```text
name
email
phone
address
city/state/zip
work authorization
availability
desired pay
resume file
LinkedIn / portfolio
```

## Automation Rule

Use this ledger to name fields once. Build a Raycast or Hammerspoon helper only
after the same fields or upload conflicts repeat across real applications.
