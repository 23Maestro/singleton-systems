# Phase One Operating System

## North Star

Build a structured business system around editing, AI-assisted workflows, proof, outreach, Upwork, finances, website copy, and packaged workflow cleanup offers.

The working lane:

> Help creators, course teams, and small video businesses turn messy video work into a clean, repeatable system.

This system exists to help Jerami stay organized, use Codex and skills on purpose, ship proof, and turn lived workflow experience into paid work.

## Current Offer

One-line offer:

> I clean up messy video workflows so projects are easier to scope, edit, review, and deliver.

Core promise:

> I help creators, course teams, and small video businesses turn messy video work into a clean, repeatable system.

Approved direction:

- Not AI-first.
- Not broad workflow consulting.
- Trusted operator voice.
- Practical, calm, useful.
- AI is useful when it reduces repetitive work.

## Phase One Goal

Create the working mold before making more tools.

Phase one is not about perfecting the final business. It is about building the repeatable system that lets us:

- write and maintain website copy
- gather proof
- document examples
- prepare Upwork assets
- run outreach
- track money and pipeline
- turn recurring solutions into future skills

## Core Surfaces

### 1. Website

Purpose:

- make the offer legible
- show proof
- explain the workflow cleanup process
- create a clear next step

Current source files:

- `docs/01_confirmed-copy.md`
- `app/page.tsx`

Use these skills when useful:

- `content-creator` for copy, sections, proof framing, and content strategy
- `sales-outreach` for pain, CTA, objection handling, and conversion logic
- `ui-designer` after the message is clear
- `frontend-developer` when implementing or polishing the site

### 2. Proof Bank

Purpose:

- turn work into evidence
- make the business believable
- feed the website, Upwork, social posts, and outreach

Every proof item should capture:

- before state
- problem
- what changed
- result
- screenshot, photo, Loom, or repo artifact
- what it proves about the offer

Proof categories:

- video workflow cleanup
- editing systems
- AI-assisted templates or summaries
- Raycast/Codex workflow builds
- client delivery systems
- small workflow demos

Passive proof assets:

- car tool inventory/mobile workflow

The car tool project is a future quick win, not the main lane.

### 3. Upwork

Purpose:

- package the offer into a buyer-friendly entry point
- create a consultative path into monthly workflow cleanup

First product:

> Video Workflow Cleanup Audit

Audit should produce:

- current workflow map
- friction notes
- quick wins
- recommended cleanup plan
- optional monthly implementation offer

Use `sales-outreach` for:

- consultation positioning
- profile language
- proposal templates
- objection handling
- follow-up messages

### 4. Outreach

Purpose:

- create conversations with creators, course teams, small video businesses, and video operators

Rules:

- keep first messages short
- lead with workflow pain, not AI
- personalize every message
- one CTA per message
- log every touch and next action

Use `sales-outreach` for:

- ICP definition
- prospect list criteria
- cold email
- LinkedIn messages
- follow-up cadence
- discovery questions

### 5. Social And Content

Purpose:

- build trust by documenting the work
- show operator thinking
- create proof before a large client base exists

Content lanes:

- messy workflow breakdowns
- before/after cleanup
- editing operations lessons
- AI-assisted workflow examples
- build-in-public notes
- practical video team systems

Use these skills when useful:

- `content-creator`
- `linkedin-content-creator`
- `instagram-curator`
- `podcast-strategist`

Default content shape:

1. What was messy.
2. What I changed.
3. Why it matters.
4. What this proves.

### 6. Finances

Purpose:

- keep the business real
- know the runway
- know what needs to close

Track:

- monthly survival number
- target monthly revenue
- active leads
- proposals sent
- booked calls
- closed work
- recurring revenue
- paid tools

Finance work should stay simple until revenue starts moving.

## Weekly Loop

Keep the loop small.

### Monday

- choose one business priority
- choose one proof asset to create
- choose one outreach target group

### Tuesday To Thursday

- build or document one concrete thing
- capture proof
- send outreach
- log responses

### Friday

- update website, Upwork, or proof bank
- review pipeline
- decide what wording the market taught us

### Weekend

- optional quick win project
- organize assets
- clean docs
- do not derail the main lane

## First Milestones

### Milestone 1: Website Copy Locked

Done when:

- hero is stable
- offer section is clear
- proof section has at least two examples
- CTA is clear

### Milestone 2: Upwork Audit Pack

Done when:

- consultation title exists
- description exists
- deliverables are clear
- discovery questions are written
- follow-up offer is defined

### Milestone 3: Proof Bank Started

Done when:

- at least three proof items are documented
- each has before/problem/change/result
- at least one has visual proof

### Milestone 4: Outreach Lane Active

Done when:

- ICP list exists
- first email exists
- follow-up exists
- simple pipeline tracker exists

## Guardrails

- Do not overbuild before proof and outreach.
- Do not make the offer AI-first.
- Do not make passive projects the main lane.
- Do not rewrite everything when a small correction is enough.
- Do not duplicate identical blocks in one response.
- Keep outputs practical and directly usable.
