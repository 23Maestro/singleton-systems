# Phase One Operating System

## North Star

Build a structured business system around workflow cleanup, repeatable systems,
portfolio, outreach, Upwork, finances, website copy, and practical AI-assisted
implementation.

The working lane:

> Help operators, creators, consultants, small teams, studios, and service businesses turn one messy recurring workflow into a cleaner system they can reuse.

This system exists to help Jerami stay organized, use Codex and skills on purpose, ship portfolio, and turn lived workflow experience into paid work.

## Current Offer

One-line offer:

> Bring one messy recurring workflow. I will help turn it into a cleaner system your team can reuse.

Core promise:

> Repeated work can be turned into cleaner, reusable systems.

Approved direction:

- Not AI-first.
- Not generic AI consulting.
- Broader than video-only.
- Trusted operator voice.
- Practical, calm, useful.
- AI is useful when it helps compare, structure, summarize, route, or speed up
  repeatable work.
- Video is the wedge. Repeatable systems are the skill.

## Phase One Goal

Create the working mold before making more tools.

Phase one is not about perfecting the final business. It is about building the repeatable system that lets us:

- write and maintain website copy
- gather portfolio
- document examples
- prepare Upwork assets
- run outreach
- track money and pipeline
- turn recurring solutions into future skills

## Core Surfaces

### 1. Website

Purpose:

- make the offer legible
- show portfolio
- explain the workflow cleanup process
- create a clear next step

Current source files:

- `docs/offer/confirmed-copy.md`
- `app/page.tsx`

Use these skills when useful:

- `content-creator` for copy, sections, portfolio framing, and content strategy
- `sales-outreach` for pain, CTA, objection handling, and conversion logic
- `ui-designer` after the message is clear
- `frontend-developer` when implementing or polishing the site

### 2. Portfolio Bank

Purpose:

- turn work into evidence
- make the business believable
- feed the website, Upwork, social posts, and outreach

Every portfolio item should capture:

- before state
- problem
- what changed
- result
- screenshot, photo, Loom, or repo artifact
- what it proves about the offer

Portfolio categories:

- video workflow cleanup
- editing systems
- AI-assisted templates or summaries
- Raycast/Codex workflow builds
- client delivery systems
- small workflow demos

Passive portfolio assets:

- car tool inventory/mobile workflow

The car tool project is a future quick win, not the main lane.

### 3. Upwork

Purpose:

- package the offer into a buyer-friendly entry point
- create a consultative path into monthly workflow cleanup

First product:

> Workflow Cleanup Buildout

The old ChatGPT project separated video editing and workflow consultation. Keep
that separation for deliverable clarity, but do not let it split the current
Singleton Systems offer. Video remains a strong portfolio lane and trust wedge, but
the offer should be able to accept one messy recurring workflow from a creator,
operator, consultant, small team, studio, or service business.

Consultation should produce:

- current workflow map
- friction notes
- quick wins
- recommended cleanup plan
- optional monthly implementation offer

Consultation page template:

1. Title
2. Category/tags
3. Description
4. What I help with
5. What the client should bring
6. Gallery asset list

Career HQ / Notion task template:

1. Goal
2. Why it matters
3. Inputs needed
4. Action list
5. Portfolio or asset link
6. Next review point

Use this pattern when a Codex or mobile Shortcut capture should become a real
Career HQ item instead of staying in Bear.

Use `sales-outreach` for:

- consultation positioning
- profile language
- proposal templates
- objection handling
- follow-up messages

Default Upwork proposal shape:

1. Pattern-interrupt question or observation.
2. One sentence showing understanding.
3. One relevant portfolio point.
4. Simple next step.

Proposal variants to generate when needed:

1. Question-first version
2. Warmer version
3. Direct version under 500 characters

### 4. Outreach

Purpose:

- create conversations with creators, course teams, small video businesses, and video operators

Rules:

- keep first messages short
- lead with workflow pain, not AI
- personalize every message
- one CTA per message
- log every touch and next action

Default LinkedIn opener:

```text
Hey [Name], quick question: are you still handling video editing / repurposing manually, or do you already have a clean process for getting clips out?
```

Use `sales-outreach` for:

- ICP definition
- prospect list criteria
- cold email
- LinkedIn messages
- follow-up cadence
- discovery questions

Content lanes:

### 6. Finances

Purpose:

- keep the business real
- know the runway
- know what needs to close

Track:

- monthly survival number
- target monthly revenue
- active leads
- proposals sent
- booked calls
- closed work
- recurring revenue
- paid tools

Finance work should stay simple until revenue starts moving.

## Weekly Loop

Keep the loop small.

### Monday

- choose one business priority
- choose one portfolio asset to create
- choose one outreach target group

### Tuesday To Thursday

- build or document one concrete thing
- capture portfolio
- send outreach
- log responses

### Friday

- update website, Upwork, or portfolio bank
- review pipeline
- decide what wording the market taught us

### Weekend

- optional quick win project
- organize assets
- clean docs
- do not derail the main lane

## Guardrails

- Do not overbuild before portfolio and outreach.
- Do not make the offer AI-first.
- Do not make passive projects the main lane.
- Do not rewrite everything when a small correction is enough.
- Do not duplicate identical blocks in one response.
- Keep outputs practical and directly usable.
