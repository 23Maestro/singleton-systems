How a real item moves

  1. Start a GitHub map when the route is too foggy or large for one session.
  2. Add children only where research, grilling, a prototype, or manual setup reveals a real branch.
  3. Link a concise Linear item when the work needs a decision, priority, ownership, or an active next move.
  4. On mobile, write Decision and Next in Linear.
  5. Assign it to Codex. Codex adds Decision / Status / Reason / Next ticket or PR to GitHub.
  6. When building starts, link Linear to the branch or pull request.

Lean guardrails

  1. Use normal professional sentences, not a Caveman persona.
  2. State outcome, needed reason, and next action.
  3. Cut filler, hedging, pleasantries, and repeated ticket context.
  4. Keep full sentences when sequence, safety, or ambiguity needs clarity.
  5. No webhook, polling job, database, or custom sync service.

Integration boundary

GitHub and Linear are connected for Singleton Systems. Use native PR and commit linking. Leave broad GitHub Issues Sync off so GitHub remains the full research and evidence record.

HTML is a scanable review surface. It does not replace the map issue.
