Harness, Not Headquarters.

This is the lightweight Singleton Systems handoff visual. It shows how a messy thought becomes a routed packet, then a task, action, portfolio item, doc rule, or memory summary without creating agents, bots, or another app.

The Flow

The harness is just the guardrail between raw intent and the surface that should actually own it.

1

Intake

Catch the thought from Bear, Raycast, Shortcuts, Codex, a link, or a portfolio asset.

2

Routing

Cerebral picks the lane, owner, packet tags, next action, and do-not list.

3

Task Shape

Opportunity HQ gets real work: status, horizon, work date, block, dependencies, portfolio notes.

4

Tool Scope

Only the tools needed for that lane are active. Raycast acts; it does not own truth.

5

Review

Drafts, writes, portfolio claims, and docs changes get checked before they count.

6

Memory

Keep the decision, owner, evidence, verification, and next action. Drop the noise.

The Surfaces

Each surface gets one job. The harness keeps them from trying to become each other.

Bear

Raw capture. Messy thoughts, quick links, someday ideas, and under-10-minute notes.

Opportunity HQ

Durable task/project state: lanes, statuses, goal horizons, work blocks, dependencies.

Raycast

Action surface at /Users/singleton23/Raycast/career-hq. Opens, logs, updates.

Eagle

Portfolio and assets. Screenshots, references, workflow examples, and evidence.

Docs + Skills

Names, routing contracts, review gates, reusable rules, and compact memory.

The Packet

This is the handoff shape. Think of it like a little routing label that travels with the work until the right owner takes over.

Not Bear tags. These labels do not create sidebar clutter.
Not a runtime format. No service reads this. It is for human/Codex clarity.
Not a new database. Opportunity HQ still owns durable task state.
[lane]Portfolio
[owner]Opportunity HQ
[intent]Package an Eagle screenshot into website/outreach portfolio.
[shape]Task with portfolio link
[tools]career-proof-packager + Eagle + Opportunity HQ; Raycast only to open/capture/update
[review]Review portfolio claim before publishing
[memory]Asset, claim, changed surface, remaining next action

The Hardening

No weak handoff: every routed item says lane, owner, shape, tools, review, and memory.
No bloat: no agents, bots, Slack, Discord, Hermes, async loops, new databases, or automation runtime.
No fake ownership: Raycast can trigger actions, but Opportunity HQ owns real task state.
No stale claims: docs/skills changes get checked with rg before they count.
