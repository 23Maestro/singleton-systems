# Singleton System Story

Date: 2026-07-21

Owner surface: Career ledger / Singleton Systems portfolio

Status: source story, not final public copy

## Core Thread

I did not start this as a developer trying to make an app.

I started unemployed, coming from broadcast tech, applying for jobs, using the new ChatGPT app for resume help, trying Upwork, and trying to build a portfolio when I did not have much to show yet.

That pressure turned into the first loop: get clearer, apply faster, make proof, learn from the next failure, and keep building.

By 2023 into 2024, the work started widening. I taught myself video editing without a formal editing career. I grew on Upwork. I was in debt. I was also traveling around 80 miles every other weekend for my kid, trying to keep life moving while figuring out how to make the work real.

In 2024, the workload got serious. I landed two real clients and started working long weeks. NurseHub was around 40 hours a week, and Prospect ID added another 20 to 25 hours. The manual work was slow, especially with a slow website, one main editor, and a growing list of recurring tasks that had to be done right.

That is where the systems thinking came from. Not from wanting to show off AI. From needing to survive the work.

One weekend, I bought and installed Keyboard Maestro, Excalibur, a Stream Deck, a Kinesis-style ergonomic keyboard, and Raycast. Raycast became the center of gravity because it matched how I wanted to work: keyboard first, fast, direct, and close to the real task.

From there I pushed ChatGPT, Claude Code, and eventually Codex into the work. I used AI the whole way, but not as a magic button. I used it as a patient technical partner while I learned the pain of asking for the wrong thing, getting boilerplate, breaking workflows, finding the edge cases, and tightening the system until it actually matched the work.

Prospect Pipeline, Scouting Command, ScoutID, the Raycast commands, the Codex environment, and the projects on GitHub all grew from that same root: real workflow pressure, repeated daily friction, and the need to make the next action faster without losing the details.

## What This Proves

This story is not "I made an AI app."

The stronger claim is:

```text
I learned the work deeply enough to know where the keyboard shortcut belongs.
```

That is the difference between a boilerplate AI demo and ScoutID/Scouting Command. The app is not just a nice interface. It comes from years of daily workflow discovery:

- which prospect detail matters while calling
- which contact action needs to happen immediately
- which panel should stay read-only
- which menu deserves a single-key command
- which automation should wait for review
- which client/contact data should be saved dynamically
- which command should stay close to Raycast instead of becoming another generic CRM screen

## Casual Public Version

I built this because the work got too real to keep doing manually.

I came from broadcast tech, was unemployed, and started using ChatGPT to help with resumes, Upwork, and portfolio ideas. I did not have a clean portfolio yet, so I had to make one while learning the work.

Then the client work hit. NurseHub was heavy. Prospect ID was heavy. I was editing, organizing, following up, checking details, and trying to move faster without dropping anything. The website was slow, the process was manual, and the edge cases kept showing up.

So I started building around the pain. Raycast became my command center. Keyboard Maestro, Excalibur, Stream Deck, and my keyboard setup became the physical layer. ChatGPT, Claude Code, and Codex became the technical partner I kept pushing until the workflows matched the real job.

ScoutID comes from that. It is not a random AI wrapper. It is a keyboard-first work surface built from years of prospect workflow details, client work, video systems, shortcut habits, and the repeated problem of needing the next action right now.

That is the story: not a developer showing off a template, but an operator who kept finding the pain, learning the system, and building until the computer started matching the work.
